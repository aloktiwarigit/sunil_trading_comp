"Workshop Almanac" — design rationale

Every shop-in-an-app I have ever seen converges on the same three aesthetic moves: bright accent colors on white backgrounds, Inter or Roboto typography, and bottom-tab navigation. None of these honor what Sunil Trading Company actually is — a multi-generational family business in a hand-painted bazaar. So this design system rejects all three.

Typography: Tiro Devanagari Hindi for display (more characterful than Noto Sans), Mukta for body (warmer than the default), Fraunces for English display (variable serif with personality), EB Garamond for English body (classic, warm). Pre-mortem failure mode #9 warned about Devanagari clipping on cheap Android — every type size and line-height in this system is sized 1.6×–1.85× line height, never less, to give Devanagari conjuncts the room they need.

Color: deep polished sheesham primary, antique brass accents (not candy gold), aged cream backgrounds (not pure white), oxblood for commitment moments. Every screen sits on a subtle paper-grain noise overlay — the texture of a 1960s shop ledger, not a 2026 SaaS surface.

Layout reframe — the Shopkeeper Presence Dock: Material 3 default is bottom-tab navigation. I'm replacing it with a top-anchored "presence dock" that ALWAYS shows Sunil-bhaiya's avatar, name in Devanagari, current status (● दुकान पर हैं / shaadi mein hain / etc.), and a quick voice-note play button. This honors the brief's "the shopkeeper IS the product" claim literally — he is structurally inescapable on every customer-facing screen.

Chat reframe — the balance scale: The chat thread renders messages with a thin brass-colored vertical thread down the center — customer messages float left, shopkeeper messages float right, the brass thread is the relationship between them. Visualizes the negotiation as a balance scale rather than a SMS-style monologue.

Curation reframe — finite, not paginated: Sally caught this in her UX Spec and I've honored it. The shortlist screen says "6 चुनी हुईं" ("6 chosen"). When you reach the bottom, there is no "load more." The finiteness IS the feature — these are Sunil-bhaiya's six picks for weddings, not the first page of a search result.

Commit reframe — Sunil-bhaiya needs your number: Pre-mortem failure mode #6 warned that OTP-at-commit framed as "verify for security" causes 64% drop-off in a relational economy. The commit screen says "सुनील भैया डिलीवरी के लिए संपर्क करेंगे" — Sunil-bhaiya will contact you for delivery. Never "verify for security." Same auth mechanism, completely different cultural framing.

Things this design system explicitly does

  • Replaces Material 3 default ColorScheme with a hand-tuned wood-and-brass palette
  • Replaces bottom-tab nav with a top-anchored Shopkeeper Presence Dock that NEVER hides Sunil-bhaiya from view
  • Renders Devanagari with line heights 1.6–1.85× (never the default 1.2×) so conjuncts breathe on cheap Android
  • Subset-ships only the Devanagari glyphs actually used by `strings_hi.dart` plus 200 most common buffer
  • Uses oxblood (#7B1F1F) ONLY for commit and udhaar acceptance moments — visually marks gravity
  • Treats the chat thread as a balance scale, not a monologue
  • Treats curated shortlists as finite, not paginated
  • Treats the OTP commit ceremony as "Sunil-bhaiya needs your number" — never "verify for security"
  • Includes a paper-grain SVG noise overlay on every page background — subtle, but anchors the aesthetic
  • Has zero hairline borders — minimum 1.5px on every divider, per the cheap-Android rendering guide
  • Voice notes use waveform bars, not progress bars — more organic, less SaaS-y
  • Brass-colored "◈" decorative dividers between sections — anchors the ledger/almanac feeling
